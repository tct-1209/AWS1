# Worklog Week 2: Amazon VPC, EC2, and Cloud Networking Basics

**Implementation period:** 27/04/2026 - 03/05/2026

## 1. Weekly objectives

- Understand core networking components: VPC, subnet, route table, Internet Gateway, and NAT Gateway.
- Practice launching Linux/Windows EC2 instances and configuring Security Groups.
- Learn how to connect to servers through SSH/RDP and validate access.

## 2. Implementation details

| Day | Date | Work details |
| --- | --- | --- |
| Monday | 27/04/2026 | Reviewed Amazon VPC architecture and noted the role of public/private subnets in system deployment. |
| Tuesday | 28/04/2026 | Studied Security Groups, Network ACLs, and proper port-opening practices for EC2. |
| Wednesday | 29/04/2026 | Launched a Linux EC2 instance and checked key pair, public IP, private IP, and SSH access. |
| Thursday | 30/04/2026 | Reviewed Windows EC2, administrator password retrieval, and RDP connection steps. |
| Friday | 01/05/2026 | Summarized common EC2 access issues such as incorrect rules, keys, or network routes. |

## 3. Outcomes

- Understood the basic network model for deploying applications on AWS.
- Learned how to secure EC2 access with Security Groups.
- Improved the ability to troubleshoot connection issues layer by layer.
