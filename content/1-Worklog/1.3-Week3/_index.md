# Worklog Week 3: Amazon RDS, Backup, and Relational Data Management

**Implementation period:** 04/05/2026 - 10/05/2026

## 1. Weekly objectives

- Study Amazon RDS and common database engines.
- Prepare VPC, DB Subnet Group, and Security Group settings for databases.
- Understand backups, snapshots, and data recovery concepts.

## 2. Implementation details

| Day | Date | Work details |
| --- | --- | --- |
| Monday | 04/05/2026 | Reviewed Amazon RDS and engines such as MySQL, PostgreSQL, SQL Server, and Aurora. |
| Tuesday | 05/05/2026 | Prepared Security Groups for EC2 and RDS, distinguishing personal-machine access from EC2-backend access. |
| Wednesday | 06/05/2026 | Created a DB Subnet Group, checked the endpoint, and documented how applications connect to a database. |
| Thursday | 07/05/2026 | Studied automated backups, manual snapshots, and recovery principles. |
| Friday | 08/05/2026 | Summarized EC2-to-RDS connectivity and common issues such as timeouts, wrong endpoints, or incorrect inbound rules. |

## 3. Outcomes

- Understood how to deploy a managed database with Amazon RDS.
- Learned how to configure secure connectivity between EC2 and RDS.
- Recognized the role of backup/snapshot in system operations.
