# Worklog Week 5: Infrastructure Automation with CloudFormation, CDK, and Cloud9

**Implementation period:** 18/05/2026 - 24/05/2026

## 1. Weekly objectives

- Understand Infrastructure as Code.
- Study CloudFormation templates, stacks, and resource deployment workflows.
- Explore Cloud9/CDK for defining infrastructure with code.

## 2. Implementation details

| Day | Date | Work details |
| --- | --- | --- |
| Monday | 18/05/2026 | Reviewed AWS CloudFormation and the role of stacks in resource management. |
| Tuesday | 19/05/2026 | Studied basic templates, parameters, outputs, and stack update processes. |
| Wednesday | 20/05/2026 | Explored drift detection to identify differences between real resources and templates. |
| Thursday | 21/05/2026 | Reviewed AWS CDK and how it defines infrastructure with programming languages instead of YAML/JSON. |
| Friday | 22/05/2026 | Explored Cloud9 as a cloud IDE for writing, running, and testing code. |

## 3. Outcomes

- Understood the benefits of Infrastructure as Code for repeatable deployments.
- Learned the main components of a CloudFormation stack.
- Built a foundation for reading and editing infrastructure templates.
