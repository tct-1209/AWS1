# Worklog Week 8: Advanced Networking with VPC Peering and Transit Gateway

**Implementation period:** 08/06/2026 - 14/06/2026

## 1. Weekly objectives

- Understand VPC Peering for connecting two VPCs.
- Study Transit Gateway as a hub for connecting multiple VPCs.
- Learn how to update route tables and test server connectivity.

## 2. Implementation details

| Day | Date | Work details |
| --- | --- | --- |
| Monday | 08/06/2026 | Studied VPC Peering, limitations, traffic flow, and use cases. |
| Tuesday | 09/06/2026 | Reviewed Cross-Peer DNS Resolution and its effect on internal name resolution. |
| Wednesday | 10/06/2026 | Studied Transit Gateway Attachments and how multiple VPCs connect to a central hub. |
| Thursday | 11/06/2026 | Reviewed route-table updates required for traffic through peering/gateways. |
| Friday | 12/06/2026 | Summarized connectivity checks using ping, private IP, route table, and Security Group. |

## 3. Outcomes

- Understood the difference between VPC Peering and Transit Gateway.
- Learned routing steps for traffic between multiple VPCs.
- Improved the ability to debug internal AWS network connectivity.
